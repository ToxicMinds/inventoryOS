InventoryOS

Enterprise Restaurant Control Built for Scale

Deterministic • Event-Sourced • Zero Vendor Lock-in

The Core Problem

Traditional restaurant tech fails at scale because they rely on mutable state.

"We lost 20 pounds of ground beef."
Standard DB updates: UPDATE inventory = inventory - 20
Without absolute auditability, trust breaks down.

The Solution: Event Sourcing

InventoryOS works like a bank ledger.

  • Never update a balance. Append an event.
  • Balances are theoretically deduced point-in-time state.
  • Complete temporal auditability built-in.

Architecture Map

Serverless deployment allows infinite scale with AWS Free Tier.

1. Storage: Single-Table DynamoDB (Zero structural drift)
2. Backend: Java 21 / Spring Boot (Deterministic engine)
3. Frontend: Next.js 14 Web / iOS / Android
4. Determinism: JVM BigDecimal Math (HALF_UP rounding)
5. Identifiers: UUIDv7 (Time-sortable causal tracing)
